## 采纳结果

唯一采纳的 `jiggle` 已合入 `task2/heuristic_v2.py`（v1 + 脚下拾取死区修补）。
**它不是策略变聪明，而是对齐连续评测环境 bait_env 的判定陷阱**：食饵弹在机器人脚下
(d==0) 时 v1 返回零动作，而 bait_env._pickups 只对"扫过的线段"判拾 → 静止拾不到脚下饵。
整秒封装版 robot_policy.py 判定是"位置相等即得分"，天然无此死区。
