# 模型赛马接入规范

## 目标

你的同学只需要交付一个可执行的 Python 脚本（单个 `.py` 文件）和对应的模型/权重文件夹。
我的电脑只会加载这个 `.py` 脚本，并由脚本自行读取它自己的权重文件夹，赛马场（我的电脑）不需要知道权重细节。

## 提交内容

- `your_model.py`：必须包含 `RobotPolicy` 类
- 可选模型文件夹：例如 `your_model_weights/`、`your_model_data/`
- 如果需要外部依赖，请尽量只使用当前环境已有的库：
  - `numpy`
  - `pandas`
  - `gymnasium`
  - `stable_baselines3`
  - `torch`
  - `matplotlib`
  - 标准库模块（`os`, `sys`, `math`, `pickle`, `json` 等）

## 运行方式

我的电脑会用下面的命令来加载你的模型：

```powershell
D:/conda/envs/AI/python.exe task2/pk_task2.py --policy path\to\your_model.py
```

你只需要保证你的 `your_model.py` 可以在我的电脑上被正常导入并实例化。

## 必须满足的接口规范

你的 `your_model.py` 内必须定义一个类：

```python
class RobotPolicy:
    def __init__(self):
        # 这里可以加载你的权重文件
        pass

    def act(self, t, active, pos=None):
        # 返回 (dx, dy)
        pass
```

### `RobotPolicy` 的要求

- `__init__(self)`
  - 不要要求额外命令行参数
  - 可以加载同目录下的模型/权重文件
- `act(self, t, active, pos=None)`
  - `t`：当前仿真时间（秒）
  - `active`：当前场上食饵列表，格式为 `[(ta, x, y, v), ...]`
    - `ta`：食饵出现时间
    - `x, y`：表格坐标，范围 `1..10`
    - `v`：食值
  - `pos`：当前机器人坐标，格式 `(x, y)`，同样是 `1..10` 范围
  - 返回值：`(dx, dy)`
    - `dx, dy` 均为整数或浮点数
    - 建议返回 `-1、0、1` 中的值
    - 最好只沿一个轴移动：例如 `(1, 0)`、`(-1, 0)`、`(0, 1)`、`(0, -1)` 或 `(0, 0)`

### 返回值说明

- `(0, 0)`：原地等待
- `(1, 0)`：向 x 正方向移动 1 格
- `(-1, 0)`：向 x 负方向移动 1 格
- `(0, 1)`：向 y 正方向移动 1 格
- `(0, -1)`：向 y 负方向移动 1 格

## 权重文件夹写法

你的模型可以和权重文件夹一起提交，例如：

```
your_model.py
your_model_weights/
    best_model.zip
    config.json
```

在脚本中推荐使用相对路径加载：

```python
import os
BASE = os.path.dirname(__file__)
model_path = os.path.join(BASE, 'your_model_weights', 'best_model.zip')
```

这样你的 `your_model.py` 在我的电脑上运行时，会正确找到它自己的权重文件。

## 推荐结构

```
your_model/
  your_model.py
  weights/
    best_model.zip
    params.json
```

或者：

```
your_model.py
weights/
  best_model.zip
```

只要 `your_model.py` 能通过自己的 `__file__` 路径定位到权重文件即可。

## 额外说明

- 不要在脚本中写死我的本地路径，例如 `E:\working_flow\program\...`
- 不要依赖交互式输入、GUI、网络请求、数据库连接等
- 不要修改 `task2/pk_task2.py`，我的电脑会直接加载你的 `.py` 文件
- 如果你想自检，可以在本地写一个简单的测试段，但提交的脚本应遵循上述接口
